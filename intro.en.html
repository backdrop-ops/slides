Why Fork Drupal?

Direction

  • When you feel like you're going the wrong direction, it's time to change paths.

Drupal 8

Drupal for the Enterprize

Drupal 8 beta is the latest, greatest release of the world's most widely used enterprise web CMS.

http://drupal.com/why-drupal-8

Marketshare

  • The larger the site, the more likely it is to be running Drupal.
  • But what about those of us who
    aren't enterprise?

Drupal Direction: Code

  • Much larger codebase (3x more Drupal code by bytes)
  • More dependencies (external libraries)
  • Most code rewritten to be Object Oriented
  • Symfony based HTTP Kernel
    • Is better(tm) software better for the whole community?

Drupal Direction: Cost

  • Drupal is free! But...
  • Specialized (expensive) developers
  • Specialized (expensive) hosting
  • Ongoing maintenance costs
  • Migration from version to version (not upgrades)
  • ICost is already a huge factor.

Decrease the complexity.

Lower the learning curve.

Shorten the time it takes to complete tasks.

Lessen the server resources required.

Decrease the cost.

A Project Management Committee (PMC)

guides Backdrop's Decision Making Process

https://backdropcms.org/leadership

  • A model successfully used by Apache projects
  • (Cassandra, Cordova, CouchDB, Hadoop, HTTP Server, Lucene, Solr, dozens others!).

The PMC's Responsibilities

  • Handles conflict resolution.
  • Sets the overall direction of the project.
  • Ensures that each change follows the project's philosophy.
  • Can change the project philosophy if necessary.
  • PMC Often (but not necessarily) core committers

In order to guide the project, the project philosophy must be defined.

Backdrop Principles

  1. Backwards compatibility is important
  2. Write code for the majority
  3. Include features for the majority
  4. Ensure Backdrop can be extended
  5. Meet low system requirements
  6. Plan and schedule releases
  7. Always remain Free & Open Source

https://backdropcms.org/philosophy

How to make web development affordable

  1. Increase the out-of-box functionality.
  2. Make maintenance easier.
  3. Always provide an upgrade path.
  4. Simplify the architecture.
  5. Consider the cost of each change
    (value backwards compatibility)
    • Make it easy on existing developers.
    • Make it fast to port modules between versions.
  6. More!

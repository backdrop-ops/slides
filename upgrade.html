What is Backdrop CMS?

Drupal 7, modernized!

  • Built-in Rich-Text editor
  • Views (and CTools) in core
  • Configuration Management System
  • Additional Field Types: Date, Email, Link

Why move to Backdrop from Drupal 7?

  • You are happy (enough) with your current Drupal 7 site
  • You want all the same features you have now
  • You want ongoing improvements to your software
  • You don't want to rebuild something that's fine
  • You want a better user/administrator experience
  • You want to save more time and/or money.
  • You agree with the Backdrop CMS Philosophy

Philosophy?

Backdrop CMS Mission

Backdrop CMS enables people to build highly customized websites affordably, through collaboration and open source software.

https://backdropcms.org/philosophy

Backdrop CMS Principles

  1. Easier updates: Backwards compatibility is important.
  2. Simplicity: Write code that the majority can understand.
  3. Focus: Only include features that benefit the majority.
  4. Extensibility: Ensure Backdrop can be customized.
  5. Security: Keep sites and people safe.
  6. Performance: Meet low system requirements.
  7. Release on-time: Plan and schedule releases.
  8. Freedom: Remain both Free and Open Source.

https://backdropcms.org/philosophy

Research what will be necessary to upgrade to Backdrop CMS

  1. Review your theme

Review your theme

Are you able and willing to port your existing theme from Drupal to Backdrop CMS?

Differences in themes

The page template (page.tpl.php) becomes
the layout template (layout.tpl.php)

Drupal 7

regions in theme
blocks in database

Backdrop CMS

regions in layouts
blocks in configuration

Minor markup changes: semantic tags, no IDs.

Drupal 7


div
id="page-title"

Backdrop CMS


article, nav, section, asside
class="page-title"

classes and attributes printed differently in templates.

Drupal 7


class="< ?php print $classes; ?>" < ?php print $attributes; ?>

Backdrop CMS


class="< ?php print implode(' ', $classes); ?>" < ?php print backdrop_attributes($attributes); ?>

functions are called from within templates

Drupal 7


print $styles;
print $scripts;

Backdrop CMS


print backdrop_get_css();
print backdrop_get_js();

Review your layouts

Review your modules

  • Custom modules
  • Contributed modules
  • Core modules

Review custom modules

  • How many do you have?
  • How complicated are they?
  • Are you able and willing to port your custom modules to Backdrop CMS?

Review core modules.

  • Have any been removed from Backdrop CMS?
  • If so, did you really need them?
  • If so, do they exist in contrib?
  • If not, are you able and willing to port them?

Review contributed modules.

  • Have any been moved into core in Backdrop CMS?
  • If some have not, do they exist in contrib?
  • If some do not, are you able and willing to port them?

Step 2:

Prepare your old Drupal 7 site

  • Ensure you are on the latest release of Drupal 7.
  • Ensure you are on the latest stable versions of all your contrib modules (and themes).
  • Delete content that you don't use/need/want anymore.
  • Uninstall modules that you don't use/need/want anymore.

Step 2:

Get Database and Files

  • Export the prepared Drupal database
  • Locate the Drupal files directory

Step 3

Prepare your new Backdrop site

  • Set up a new Backdrop codebase
  • Add all custom and/or contributed modules
  • Add all ccustom and/or contributed layouts
  • Add all custom and/or contributed themes

Step 3

Load Database and Files

  • Import the prepared Drupal database
  • Copy the Drupal files directory

Step 4

Upgrade

(visit update.php)

Step 4

QA and adjust

  • Confirm that everything is working.
  • Adjust configuration as needed (commit to vcs).
  • Repeat steps 3-4 as needed, sync config.
